# benchmark-mc

**Behavioral benchmark MCQ environment for testing LLM overgeneralization of library rules.**

Every question presents a code snippet where a known rule *almost* applies — but doesn't, due to a non-obvious boundary condition. The hard negative distractor is always the output a model confidently applying the rule would predict, making wrong answers a signal of systematic overgeneralization rather than mere knowledge gaps.

All questions are **execution-verified**: both the confirming snippet and the confounder were actually run before inclusion.

## Benchmarks included

| Library | Tasks | Families |
|---------|------:|---------|
| pandas | 10 | groupby_semantics, dtype_coercion, nan_semantics, index_alignment, copy_semantics |
| scikit-learn | 10 | preprocessing_scaling_behavior, cross_validation_data_leakage, feature_importance_interpretation, random_state_reproducibility |

## Example question

```
Rule: StandardScaler applies training mean/variance to test data.

Question — what does this print?

  scaler = StandardScaler()
  scaler.fit([[10],[20],[30]])
  scaler.fit([[100],[200],[300]])   ← called fit() again
  print(scaler.transform([[100]])[0][0])

  A. -1.224744871391589   ← CORRECT (second fit() overwrites statistics)
  B. -1.0
  C. 0.0
  D. 9.797958971132712    ← hard negative (rule naively applied)
```

## Usage

```bash
# Evaluate a model on all 20 questions
prime eval run benchmark_mc -m openai/gpt-4.1-mini -n 20

# Target one library
prime eval run benchmark_mc -m openai/gpt-4.1-mini --env-args '{"library": "pandas"}' -n 10

# Evaluate a small open model (via hosted inference)
prime eval run benchmark_mc -m openai/Qwen/Qwen2.5-1.5B-Instruct -n 20 --hosted
```

## For RL training

```python
from benchmark_mc.benchmark_mc import load_environment

# Chain-of-thought mode recommended for RL — rewards structured reasoning
env = load_environment(library="all", chain_of_thought=True)
```

The environment returns `1.0` for a correct letter answer, `0.0` otherwise. The model can reason freely; only the last standalone `A/B/C/D` on the final line is scored.

To bootstrap training on models that initially abstain, set `format_reward_weight=0.2` for the first ~50 gradient steps, then anneal to `0.0`.

## Generating more questions

The environment is generated by [benchmark-creator](https://github.com/dinkarjuyal/benchmark-creator). To add more questions or target a new library:

```bash
ANTHROPIC_API_KEY=sk-ant-... python3 -m benchmark_creator \
  --repo https://github.com/your-library/repo \
  --strategy sgs --n 5 \
  --output benchmarks/your_library

python3 scripts/bundle_pi_data.py --benchmarks your_library
prime env push --auto-bump
```
