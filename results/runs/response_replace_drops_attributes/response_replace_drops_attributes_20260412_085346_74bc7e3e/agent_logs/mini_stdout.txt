class Response(object_ref):
    """An object that represents an HTTP response, which is usually
    downloaded (by the Downloader) and fed to the Spiders for processing.
    """

    attributes: Tuple[str, ...] = (
        "url",
        "status",
        "headers",
        "body",
        "flags",
        "request",
        "certificate",
        "ip_address",
        "protocol",
    )
    """A tuple of :class:`str` objects containing the name of all public
    attributes of the class that are also keyword parameters of the
    ``__init__`` method.

    Currently used by :meth:`Response.replace`.
    """

    def __init__(
        self,
        url: str,
        status=200,
        headers=None,
        body=b"",
        flags=None,
        request=None,
        certificate=None,
        ip_address=None,
        protocol=None,
    ):
        self.headers = Headers(headers or {})
        self.status = int(status)
        self._set_body(body)
        self._set_url(url)
        self.request = request
        self.flags = [] if flags is None else list(flags)
        self.certificate = certificate
        self.ip_address = ip_address
        self.protocol = protocol

## Problem
The `Response.replace()` method was not preserving attributes from the original 
response. When calling `resp.replace(status=404)`, the resulting response would 
have default values for url, headers, body, etc., instead of copying them from 
the original response.

## Root Cause
The `replace()` method in `scrapy/http/response/__init__.py` had a loop that 
iterated over `self.attributes` but did nothing with them:

```python
def replace(self, *args, **kwargs):
    """Create a new Response with the same attributes except for those given new
values"""
    for x in self.attributes:
        pass  # BUG: attributes never carried over
    cls = kwargs.pop("cls", self.__class__)
    return cls(*args, **kwargs)
```

This meant that when creating a new Response instance, only the kwargs 
explicitly provided to `replace()` were passed, leaving required positional 
arguments (like `url`) missing, causing `TypeError`.

## Solution
Modified the `replace()` method to copy all attributes from `self` into `kwargs`
if they are not already provided:

```python
def replace(self, *args, **kwargs):
    """Create a new Response with the same attributes except for those given new
values"""
    for x in self.attributes:
        if x not in kwargs:
            kwargs[x] = getattr(self, x)
    cls = kwargs.pop("cls", self.__class__)
    return cls(*args, **kwargs)
```

This ensures:
1. All attributes from the original response are preserved unless explicitly 
overridden
2. Explicitly provided kwargs override the original values
