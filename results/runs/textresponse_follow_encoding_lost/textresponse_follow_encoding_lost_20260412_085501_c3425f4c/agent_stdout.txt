def _url_from_selector(sel):
    # type: (parsel.Selector) -> str
    if isinstance(sel.root, str):
        # e.g. ::attr(href) result
        return strip_html5_whitespace(sel.root)

The `TextResponse.follow(url)` method was not properly handling relative URLs. 
After a recent change, the `urljoin` call was removed, causing relative hrefs

In the `TextResponse.follow()` method in `scrapy/http/response/text.py`, there 
was a bug on line 198:
```python
encoding = None  # BUG: encoding always None, ignores response charset
```

This line was unconditionally setting `encoding = None`, overriding any value 
that was passed as a parameter or defaulting from the parent class. When this 
None value was passed to the `Request` constructor, it caused a TypeError 
because the `safe_url_string()` function requires a string encoding.

## Solution
The fix was to replace the buggy line with proper encoding handling:

```python
# BEFORE (buggy)
encoding = None  # BUG: encoding always None, ignores response charset

# AFTER (fixed)
if encoding is None:
    encoding = self.encoding
```

This change ensures that:
1. If the user provides an `encoding` parameter, it's used
2. If no `encoding` is provided (the default is None), the response's encoding 
is used
